## 🔥 Featured Project — AWS Three-Tier Architecture

<div align="center">
  <a href="https://github.com/aadityapalsagwan/three-tier-architecture-aws-ssm">
    <img src="https://github-readme-stats.vercel.app/api/pin/?username=aadityapalsagwan&repo=three-tier-architecture-aws-ssm&theme=github_dark&hide_border=true&show_owner=false" />
  </a>
</div>

<br/>

> 🏗️ Designed and deployed a **secure, scalable, and highly available** 3-tier architecture on AWS using industry best practices — **zero SSH access**, no hardcoded credentials, fully isolated network layers.

```
Internet
    │
    ▼
┌─────────────────────────────────────┐
│     Application Load Balancer        │  ← PUBLIC SUBNET
│     (ALB - Traffic Distribution)     │
└─────────────────┬───────────────────┘
                  │
┌─────────────────▼───────────────────┐
│        WEB + APP TIER (EC2)          │  ← PRIVATE SUBNET
│        Node.js | Nginx               │
│        Access via SSM (No SSH)       │
└─────────────────┬───────────────────┘
                  │
┌─────────────────▼───────────────────┐
│         DATABASE TIER                │  ← ISOLATED SUBNET
│         Amazon Aurora (Multi-AZ)     │
└─────────────────────────────────────┘
```

| ✅ Feature | 🔧 How It's Done |
|:---|:---|
| Zero SSH Access | AWS SSM Session Manager |
| No Hardcoded Secrets | IAM Roles & Instance Profiles |
| High Availability | Multi-AZ Deployment |
| App Deployment | S3 Bucket |
| Network Isolation | VPC + Public/Private Subnets |
| Web Server | Nginx (Reverse Proxy) |
